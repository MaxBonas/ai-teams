# AI Teams v0.1.0

Primera candidata de distribución del control plane AI Teams reconstruido sobre
SQLite. La identidad exacta de la publicación es el tag anotado `v0.1.0`; el
manifiesto del paquete conserva además el commit de origen.

## Novedades

- Flujo Lead-first con issues, runs, heartbeats, delegaciones e interactions
  durables.
- Perfiles canónicos `solo_lead`, `lead_quorum` y `full_team`, con gates
  proporcionales al riesgo.
- Catálogo de proveedores/modelos y ranking por rol separado del estado de
  configuración local de cada adapter.
- Bootstrap reproducible con locks Python/npm, diagnóstico de máquina y soporte
  explícito de toolchains.
- Artefacto derivado solo de Git, sin runtime local, con SHA-256, manifiesto,
  CycloneDX y reporte de licencias.

## Instalación y actualización

Una instalación nueva debe seguir
[`INSTALLATION_AND_INTEGRATION.md`](../INSTALLATION_AND_INTEGRATION.md). Una
instalación existente debe seguir el procedimiento side-by-side de
[`UPGRADE_AND_ROLLBACK.md`](../UPGRADE_AND_ROLLBACK.md), verificar primero el
checksum externo e interno y no copiar `venv/`, `node_modules/`, `.env` ni
`runtime/` desde otra máquina.

## Migración y rollback

El migrador canónico es `scripts/migrate_to_v2.py`. Se ejecuta primero en
dry-run y después con `--apply`, conservando el backup que crea por defecto.
Volver al código anterior después de una escritura de esquema exige detener el
sistema y restaurar ese backup; cambiar solo el código no constituye rollback.

## Límites conocidos

- La aceptación real verificada sigue limitada a Windows x86-64. Linux, macOS,
  ARM64 y cada adapter vivo conservan gates independientes.
- Las sesiones, logins y API keys de proveedores son configuración local y no
  se incluyen en el paquete.
- Ollama y LM Studio son opcionales; no son requisitos básicos.
- La compatibilidad upstream entre Starlette y httpx 2 permanece vigilada como
  `RUN-020`; no se fuerza una actualización incompatible.
- Esta candidata no se publica hasta que el mismo ZIP promocionable complete la
  aceptación independiente y multiplataforma I.8.4.
